# 质量契约（Phase 1 产出）

> 本文件由 Phase 1 契约骨架阶段产出。定义本项目的**质量验证标准与自动化检查**。
> 这是 Phase 2 所有 Batch 的验收依据——质量契约是真相源，静态文档不是。

---

## 0. 质量契约是什么

质量契约 = 一组**可自动验证**的质量规则。每条规则必须：
- 有明确的检查方式（工具/命令/脚本）
- 有明确的通过标准
- 在每 Batch 末尾执行

质量契约不是"建议"或"最佳实践"——它是**强制验收门**。

---

## 1. 质量维度

根据项目领域，选择适用的质量维度：

| 维度 | 说明 | 适用领域示例 |
|------|------|------------|
| 一致性 | 风格/格式/命名是否统一 | 所有领域 |
| 完整性 | 是否缺少必要的部分 | 所有领域 |
| 正确性 | 事实/数据/逻辑是否正确 | 写作/研究/代码 |
| 可追溯性 | 决策/引用是否有来源 | 研究/设计/代码 |
| 合规性 | 是否符合外部标准/法规 | 设计/代码/视频 |
| 性能 | 是否满足性能指标 | 代码/视频 |

## 4. 质量契约演进

- Phase 2 期间：每 Batch 可追加新规则
- Phase 3 期间：规则稳定，修改需立决策记录
- 任何规则的**放宽**必须立决策记录说明理由

---

## 5. 如何使用本模板

1. 根据项目领域选择相关的质量维度
2. 每个维度下列 2-5 条具体规则
3. 每条规则必须有"检查方式"和"通过标准"——不能用主观描述
4. Phase 1 初版只需覆盖最关键的 5-8 条规则
5. Phase 2 执行过程中逐步补充
